Definition

VCS (Version Control System) - is a system that records changes to a file or set of files over time so that you can recall specific versions later.

Analogue of VCS in Russian: система контроля версий

.

It worth noting that professional programmers use one of available VCS. Almost always they use Git - it's de-facto the standard in programming community, especially in open-source projects.

Advantages of usage of VCS

Using VCS has several advantages:

  1. Chiefly it helps teams of programmers maintain project live-cycle. Every programmer has his own up-to-date copy of project, he can make any changes and he can restore changes if needed (and return to previous working version :) ).
  2. You always have reassuring feeling that your work wont disappear due to some changes or hard disc breakdown.
  3. You don't need to track in which folder do you have to store the 0.3.0.1 version of your project - it's matter of VCS.
  4. Your manual work comes down to typing required commands in VCS.

Web-hostings of VCS

Some VCS (such as Git) has hostings. The largest hosting of Git is GitHub - website, where millions of programmers all over the planet store their projects their.
