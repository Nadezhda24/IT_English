Definition

Garbage collector (GC) - is an instrument for automatic memory management. It's presented in Python, Java, C# and other programming languages.

Analogue of garbage collector in Russian: сборщик мусора

.

How does it work?

If piece of memory is no longer referenced by any variable in program it's called garbage. GC periodically tries to reclaim such memory to free memory for future use.

As you may know the capacity of RAM of computer is limited. If program will not release useless memory we will get memory leak. Memory leak is a situation when a program loses ability to reuse some region of memory due to deletion of reference or pointer to that region.

In programming programming languages with manual memory management such situations aren't unusual. For example in C and C++ developer can simply forget to release dynamically allocated memory when it becomes useless. Such program gradually consumes all available memory of computer. At the end that program will break down system, because other programs won't be able to get required memory.
