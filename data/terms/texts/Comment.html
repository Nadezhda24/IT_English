Definition

In computer programming, a comment is a human-readable explanation in the source code of a computer program. Developers adds comments with the purpose of making the source code easier for humans to understand, and are generally ignored by compilers and interpreters.

Comments are sometimes also processed in various ways to generate documentation external to the source code itself by documentation generators, or used for integration with source code management systems and other kinds of external programming tools. For example every mature IDE maintains the IntelliSense technology which allows a programmer to get a documentation attached to a particular function (if it presented).

Analogue of comment in Russian: комментарий.

Comment syntax

The syntax of comments in various programming languages varies considerably. Here is a brief list of what you may faced with:

  • In C and C++ you can make a comment by printing // - single line comment or /**/ - multiline comment, which apparently can span multiple lines in a source code.
  • In Assembler you can create a comment with help of semicolon symbol ; which is normally used to separate statements in compiled programming languages.

Some IDEs support so-called documenting comments which take the form of /// or /**.

Pitfalls of using comments

Newcomer programmers have two extremeness

  • They don't write comments at all
  • They try to comment every line and even obvious code

Remember that you should use comments wisely, because they are intended for clarifying ambiguities.

Famous proverb and truthful

Code never lies, comments sometimes do.
