Definition

Socket - a program interface for sending and recieving data between two computers through the network.

Analogue of Network socket in Russian: Сокет.

Server and client sockets

There are 2 types of sockets:

  • Server sockets - a type of socket that waits for requests to come in over the network. When server socket gets a connection, it provide an instance of client socket that bound with remote endpoint (IP-address and port of connected client). After that, server can use this connection for its intended purpose.
  • Client sockets - a type of socket that start a connection between client and server. Client socket use a ip-address and port of the remote host to establish a connection. After it has been done, programmer can use this interface to proceed data through it and waiting for the answer.

Connection protocols

There are two most common protocols of transmitting information through the network - UDP and TCP.
Sockets using UDP called connectionless sockets or datagramm sockets (because UDP - User Datagramm Protocol). Datagramm sockets can't guarantee safety of transmitting information but they are very fast.
Sockets using TCP calles connection-oriented sockets or stream sockets. Stream sockets monitor the safety of transmitting information but it leads to an increase of network traffic. Also, TCP socket is slower than UDP one.
