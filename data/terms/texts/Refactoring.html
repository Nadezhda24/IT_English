Definition

Refactoring - the process when a programmer tries to make a part of an existing code base a bit more tidy and obvious. Moreover, the process of refactoring can be tied with another process - optimization.

In other words, refactoring - is an attempt of rewriting an existing code chunk to get more clear results.

Analogue of refactoring in Russian: рефакторинг

Why should developers make a refactoring from time to time?

There're some key points of why should we make our code bases cleaner:

  • Usually there're programmers with different skills of writing code and understanding "what's going on". Entry-level engineers usually unable to write a code of a good quality. After several month of high-load work on a project some parts of a code base may look like strange and useless (but this it's only your feeling of it!).
  • Everyday different programmers do changes on existing code. Their skills gradually increase during their career. At some moment they can consider a previous code chunks as awful ones or even disgusting.
  • Due to everyday code changes project becomes dirty and can't be understood fully by one person (except the teamlead, maybe). Programmers have to control a complexity of the software they produce.
  • Your elder teammates can improve some parts of your previous code for suiting team's conventions or naming style.

Some tips for making refactoring

Keep changes atomic and small. Really, developer shouldn't try to change all at once. With a high probability such programmer will break even perfectly working parts of a software.

Always use CVS when you make a refactoring to avoid previous situation - you will have an opportunity to revert changes back.
