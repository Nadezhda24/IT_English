Definition

In computer programming, a handle is an abstract reference to a resource that is used when application software references blocks of memory or objects that are managed by another system like a database or an operating system.

Analogue of handle in Russian: дескриптор

What is a handle internally?

In many systems handles are just integers. They have no meaning outside of the system that created it. Hence only a parent-system knows what each handle points to.

Where can you face with handles?

At the time of writing this article the most popular and widespread operating system is Windows. Windows has it's own API(Application programming interface - the set of functions that can be used by programmers in building their Windows-programs.

For instance, when a programmer create a window (rectangle area of screen that can receive user input and can output graphic information) using Windows API he invokes a functions CreateWindow. That function returns a handle to a Window - a specific reference to a particular resource in operating system, in this case - a reference to created window.

Let's investigate another example. Imagine that you develop a network application for Linux - your own Telegram. To make a connection between host and a client of your service you have to use sockets - it's an abstraction of a data-transfer connection between two programs. Insofar as a socket is a resource of the operating system we can refer a particular socket using it's handle
